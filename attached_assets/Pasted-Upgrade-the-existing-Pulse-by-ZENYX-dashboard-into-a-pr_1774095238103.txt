12) BACKEND / LIVE DATA FIX
There is an important issue:
When the client firewall was restarted, Zabbix showed the status change, but the frontend did not update.
This must be fixed properly.

Implement the correct backend flow:

Frontend:
- poll backend every 5 seconds
- show loading / refreshing state
- update UI immediately when status changes

Backend:
- fetch fresh data from Zabbix on every polling cycle
- disable caching for monitoring endpoints
- map Zabbix triggers / hosts / problems into frontend-friendly JSON
- ensure latest status is always returned
- include last_updated timestamp
- include business impact mapping

Possible cause right now:
- stale cache
- frontend not polling
- middleware not re-fetching
- wrong trigger mapping
- frontend state not re-rendering after API response

Fix this properly.
